================================================================================
DATABASE SCHEMA (SUPABASE/POSTGRESQL)
================================================================================

Tables:

1. profiles
   - id (uuid, primary key)
   - user_id (uuid, references auth.users)
   - full_name (text)
   - username (text, unique)
   - tanggal_setor (date, nullable)
   - created_at (timestamp)
   - updated_at (timestamp)

2. user_roles
   - id (uuid, primary key)
   - user_id (uuid, references profiles.user_id)
   - role (text) - values: 'admin', 'mitra', 'user'
   - created_at (timestamp)

3. tagihan (bills)
   - id (uuid, primary key)
   - user_id (uuid, references profiles.user_id)
   - nama (text) - nama tagihan/pelanggan
   - jumlah (integer) - jumlah tagihan
   - status (text) - 'belum_lunas', 'lunas'
   - nama_input (text) - user yang input
   - nama_lunas (text, nullable) - user yang mark sebagai lunas
   - created_at (timestamp)
   - updated_at (timestamp)
   - deleted_at (timestamp, nullable) - untuk soft delete

4. settings
   - id (uuid, primary key)
   - user_id (uuid, references profiles.user_id)
   - saldo_awal (integer)
   - created_at (timestamp)
   - updated_at (timestamp)

5. profit_settings
   - id (uuid, primary key)
   - profit_percentage (integer)
   - created_at (timestamp)
   - updated_at (timestamp)

6. pending_registrations
   - id (uuid, primary key)
   - full_name (text)
   - username (text, unique)
   - password_hash (text)
   - status (text) - 'pending', 'approved', 'rejected'
   - reviewed_at (timestamp, nullable)
   - created_at (timestamp)

7. notifikasi_user
   - id (uuid, primary key)
   - title (text)
   - message (text)
   - target_type (text) - 'all', 'specific'
   - target_user_id (uuid, nullable)
   - is_read (boolean)
   - read_at (timestamp, nullable)
   - created_at (timestamp)

8. app_settings
   - id (uuid, primary key)
   - key (text) - e.g., 'app_name'
   - value (text)
   - created_at (timestamp)
   - updated_at (timestamp)

================================================================================
FITUR UTAMA APLIKASI
================================================================================

1. SISTEM AUTENTIKASI
   - Login dengan email/username dan password
   - Registrasi user baru (dengan approval admin untuk reseller)
   - Role-based access control:
     * Admin: Akses penuh ke semua fitur
     * Mitra (Reseller): Akses terbatas untuk manajemen tagihan
     * User: Akses dasar untuk melihat tagihan pribadi
   - Logout dengan session management
   - Persistent login dengan localStorage

2. MANAJEMEN TAGIHAN (BILLS)
   - Tambah tagihan baru (AddTagihan)
   - Tambah tagihan langsung lunas (BayarLangsung)
   - Mark tagihan sebagai lunas
   - Update tagihan (edit nama dan jumlah)
   - Soft delete tagihan (dengan restore)
   - Permanent delete tagihan
   - Hapus semua tagihan lunas sekaligus
   - Filter dan search tagihan
   - Kategorisasi:
     * Tagihan Aktif (belum lunas)
     * Tagihan Lunas
     * Tagihan Terhapus (deleted)

3. SISTEM INVOICE
   - Generate invoice untuk tagihan
   - Format invoice profesional dengan html2canvas
   - Download/print invoice
   - InvoiceDialog component untuk preview

4. DASHBOARD
   - Dashboard Cards menampilkan:
     * Total Tagihan
     * Total Lunas
     * Total Belum Lunas
     * Sisa Saldo
   - Welcome greeting dengan nama user
   - Statistik visual dengan charts (Recharts)
   - Real-time updates

5. FITUR KHUSUS ADMIN
   - User Management:
     * Tambah user baru
     * Edit user (nama, role, password)
     * Hapus user
     * Assign role (admin, mitra, user)
     * Set permissions (edit, delete, mark lunas)
   - Pending Registrations:
     * View pendaftaran reseller baru
     * Approve/reject pendaftaran
     * Auto-create Supabase auth user pada approval
   - Tanggal Setor Mitra:
     * Set tanggal setor untuk mitra
     * Edit tanggal setor
   - Pesan ke User:
     * Kirim notifikasi ke semua user
     * Kirim notifikasi ke user spesifik
     * History pesan
   - Rekap Laporan:
     * Generate laporan
     * Export data
   - Hapus Data:
     * Bulk delete data
     * Cleanup data lama
   - App Name Settings:
     * Customizable nama aplikasi
     * Global app settings

6. FITUR KHUSUS MITRA (RESELLER)
   - MitraDashboard dengan fitur khusus
   - Manajemen tagihan sendiri
   - View deposit date
   - Profit settings

7. FITUR PUBLIK
   - Cek Tagihan:
     * Search tagihan berdasarkan nama
     * View status pembayaran
     * Tidak perlu login
   - Reseller Registration:
     * Form pendaftaran reseller
     * Menunggu approval admin

8. NOTIFIKASI
   - Real-time notifications dengan Supabase realtime
   - NotifikasiPopup untuk popup notifikasi
   - Mark as read functionality
   - Auto-delete notifikasi lama (24 jam)

9. PROFIL & PENGATURAN
   - Deposit date management
   - Profile information
   - App settings context

10. AUTO CLEANUP
    - Auto-delete soft-deleted tagihan > 24 jam
    - Auto-delete read notifications > 24 jam
    - Runs periodically

11. PERMISSIONS
    - Role-based permissions:
     * canEditData
     * canDeleteData
     * canLunasData
    - Configurable per user

12. PROFIT TRACKING
    - Profit percentage settings
    - Profit calculations
    - Global profit management
